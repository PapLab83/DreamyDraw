# REQUIREMENTS.md - Спецификация требований

## 1. Функциональные требования
- **Режимы работы:**
    - `fast`: Генерация всего контента за один проход.
    - `check`: Остановка после генерации текста для каждой истории. Требуется явное подтверждение пользователя (через CLI или Web) для продолжения (генерации картинки).
- **Парсинг ввода:** Система должна извлекать количество иллюстраций из текстового запроса (например, "3 лисы" -> count=3). По умолчанию 3.
- **Генерация контента:**
    - Текст: 3–5 предложений, адаптированных под возраст 3–5 лет.
    - Картинка: Генерация промпта включает текст истории для наложения.
    - Вопросы: 2–3 простых вопроса для обсуждения с ребенком.
- **Кеширование:** Хеш ключа (тема + режим + стили + количество) используется для поиска в `cache.json`. Если найдено — возвращается готовый результат.

## 2. Нефункциональные требования
- **Хранение данных:** Только JSON файлы.
- **Блокировки:** Использование `portalocker` для обеспечения атомарности записи в JSON.
- **Расширяемость:** Добавление новых стилей или миров не должно требовать изменения класса `Orchestrator`.
- **Постобработка (План Б):** Если провайдер картинок не поддерживает наложение текста или делает это плохо, система должна иметь возможность переключиться на ручное наложение через Pillow.

## 3. Ограничения
- Минимальный возраст ребенка: 3 года.
- Максимальное количество иллюстраций за раз: 10 (ограничение для защиты от спама/ошибок).
- Формат изображений: JPG.
