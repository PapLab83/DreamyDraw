# План реорганизации валидации и редактуры (Refinement)

Этот документ описывает переход от полной перегенерации плана к точечной редактуре проблемных тем.

---

## 1. Новая структура узлов (Nodes)

- **step-series-planner:** Создает первичный план (только при первом запуске).
- **step-plan-validator:** Проверяет план, выдает адресные замечания по конкретным темам.
- **step-plan-refine (НОВЫЙ):** Принимает текущий план + замечания и исправляет ТОЛЬКО проблемные пункты, сохраняя удачные.

---

## 2. Форматы взаимодействия

### Валидатор (PLAN_VALIDATOR.md)
Должен возвращать JSON, где в причинах и советах явно упоминается название темы.
- *Пример:* `"reason": "В теме 'Рыжик на реке' описание 'ловит рыбу сетью' не подходит для режима Правда."`

### Редактор (REFINE_PROMPT)
Инструкция: "Ты — редактор. Тебе дан план. Темы [0, 2] одобрены. Тема [1] требует правок: [замечание]. Верни полный план, где изменены только темы с замечаниями."

---

## 3. Логирование
Для каждого шага выводить:
- Заголовок темы.
- Содержание (content).
- Статус (Approved / Rejected).
- Адресный совет, если Rejected.
