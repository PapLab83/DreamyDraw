# Спецификация оркестратора (DreamyDraw)

Этот документ является техническим описанием логики работы пайплайна DreamyDraw. Он служит основой для реализации Этапа 5 ROADMAP.

---

## 1. Общий алгоритм

Оркестратор работает по принципу **Plan-Execute-Validate**. Он управляет состоянием сессии и координирует вызовы провайдеров. Поддерживает как одиночные рассказы, так и серии (связанные истории про одного персонажа).

---

## 2. Детализация шагов пайплайна

### step-safety-gate [#step-safety-validate]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Проверить тему на безопасность, цензуру и соответствие детской ЦА.
- **Логика:**
    - Использовать LLM для анализа `topic`.
    - Если тема содержит насилие, "взрослый" контент или явно опасные сценарии — остановка с ошибкой.
- **DoD:** Тема признана безопасной.

### step-config-match [#step-config-match]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Проверить логическую совместимость темы и выбранного режима правдивости.
- **Логика:**
    - Если `truth_mode = TRUTH` (Правда), а в теме — магические существа (например, "ковер-самолет"), ИИ должен предложить сменить режим или скорректировать ожидания.
    - Выбор: продолжить (если совместимо) или предложить пользователю уточнение.
- **DoD:** Конфигурация признана логически верной для данной темы.

### step-series-planner [#step-series-planner]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Создать связный план для N историй без повторов.
- **Логика:**
    - Если `count > 1`, LLM генерирует N уникальных аспектов темы (например: сон, еда, игра).
    - Формирует `global_context` (описание персонажа, чтобы он был одинаковым во всех историях).
    - Если `count = 1`, план состоит из одного пункта (самой темы).
- **DoD:** Массив `series_plan` в `SessionState` заполнен.

### step-idea-generator [#step-idea-generator]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Генерация пула сюжетов для конкретного шага серии.
- **Логика:**
    - Для текущего аспекта из `series_plan` генерирует 10 идей.
- **DoD:** Массив `raw_ideas` готов.

### step-idea-scoring [#step-idea-scoring]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Присвоить каждой идее "Детский индекс".
- **Логика:**
    - Оценка каждой идеи от 0 (опасно) до 1 (идеально для 3-5 лет).
- **DoD:** Каждая идея имеет `child_index`.

### step-score-normalize [#step-score-normalize]
- **Тип:** ⚙️ Детерминированный
- **Цель:** Приведение индексов к распределению для корректного выбора.
- **Логика:**
    - Функция нормализации весов (softmax или простая линейная).
- **DoD:** Идеи имеют веса для сэмплинга.

### step-idea-sampler [#step-idea-sampler]
- **Тип:** ⚙️ Детерминированный
- **Цель:** Случайный выбор одной идеи из пула.
- **Логика:**
    - Взвешенная случайная выборка на основе нормализованных индексов.
- **DoD:** Выбрана финальная идея для этого шага.

### step-child-filter [#step-child-filter]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Генерация текста истории и вопросов на основе выбранной идеи.
- **Логика:**
    - Применяются Markdown-промпты (Base + Truth + Style + Global Context).
- **DoD:** `StoryItem` содержит `text` и `questions`.

### step-story-validator [#step-story-validator]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Проверка готовых текстов историй на соответствие режиму и качеству.
- **Логика:**
    - После генерации всех текстов серии, каждый текст проверяется Критиком.
    - Критик ищет антропоморфизм, ошибки в фактах или избыточную сложность.
- **DoD:** Тексты признаны качественными или отправлены на редактуру.

### step-story-refine [#step-story-refine]
- **Тип:** 🤖 ИИ (LLM)
- **Цель:** Исправление текстов историй по замечаниям Критика.
- **Логика:**
    - До 3-х попыток исправления текста.
- **DoD:** Текст истории исправлен.

### step-image-pipeline [#step-image-pipeline]
- **Тип:** 🤖 ИИ (Image Model)
- **Цель:** Генерация иллюстрации.
- **Логика:**
    - Сборка image-промпта (Base + Style + StoryText).
    - Асинхронный вызов провайдера.
- **DoD:** Файл сохранен в `output/`.

## 3. Конфигурация оркестрации

| Параметр | Значение | Описание |
| :--- | :--- | :--- |
| `MAX_STORY_RETRIES` | 3 | Максимум правок от валидатора |
| `POLLING_INTERVAL` | 15s | Частота обращения к Media API |
| `PLANNER_BATCH_SIZE` | 10 | Количество идей для скоринга |

---

## 4. Масштабируемость (LangGraph)
Каждый `step-*` проектируется как независимая функция (Node), принимающая и возвращающая состояние, что обеспечит бесшовный переход на LangGraph.
