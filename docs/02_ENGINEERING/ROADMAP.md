## Тех. долг (заметки)

### 🔧 Предсуществующие баги (не блокеры миграции — выносим)

**1. CLI: `topic` обязателен даже с `--session`**
- `cli_parser.py`: `nargs="?"` на `topic`
- `main.py`: явная проверка `not args.session and not args.topic` → `parser.error(...)`
- Тест: `python main.py --session <id>` без topic

**2. `LLMMockProvider` возвращает один и тот же текст на любой промпт**
- Не отдаёт JSON для safety/config/planner/scoring/validator
- Решение: переделать в роутер по сигнатуре промпта — детектить "является ли промптом X" по ключевым словам и отдавать заглушку нужного формата
- Альтернатива: набор `mocks/*.json` файлов + match по regex от промпта
- Без этого `--mode fast` без реальной LLM нерабочий

**3. Langfuse `Context error: No active span` (warnings)**
- `Orchestrator._update_trace_metadata` вызывается до `graph.invoke` (нет активного span)
- `_extract_interrupt` вызывается после закрытия span
- Решение А: убрать `_update_trace_metadata` — каждая нода уже сама создаёт span через `@observe`
- Решение Б: `logging.getLogger("langfuse").setLevel(logging.ERROR)` в `logging_config.py`
- Рекомендую А (чистая архитектура) + Б (на случай других warnings)

### ⚠️ Побочные эффекты миграции (рекомендую закрыть сейчас)

**4. `print_final` неверно интерпретирует `failed`**
- При отмене пользователем (`n` в CHECK) пишет "остановлена из-за ошибки"
- Это **прямое следствие** того, что мы используем `current_node = "failed"` для двух разных вещей: реальная ошибка и пользовательская отмена
- Решение А: разделить через эвристику (`if any(s.text for s in stories)` → "отменено")
- Решение Б: ввести отдельный `cancelled` статус (более чистый, но трогаем схему)
- Рекомендую А — короткая правка только в `main.py`

**5. Restore из `--session` после Ctrl+C — не протестирован**
- MemorySaver между процессами теряется → граф должен восстановиться по `current_node` из JSON
- Сейчас работает только потому, что **первая нода после restore читает `current_node` из state и принимает решения** (например, `plan_validator` смотрит на `approved_indices`)
- Но точки **interrupt** (например, прервали Ctrl+C на `user_confirmation`) — не восстановятся, т.к. MemorySaver пустой
- Решение: добавить в `Orchestrator.run_pipeline` логику "если в state есть pending interrupt-like состояние, начать с правильной ноды через entry_point_from_session"
- **Это критично для UX**, но требует отдельной аккуратной работы
- В этап 6 (тесты) можно включить тест-кейс restore, чтобы зафиксировать дыру
