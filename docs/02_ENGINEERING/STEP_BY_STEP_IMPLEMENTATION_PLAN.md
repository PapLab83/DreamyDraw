# План реализации Оркестратора (Этап 5) — ВРЕМЕННЫЙ РАБОЧИЙ ДОКУМЕНТ

Этот документ является руководством по реализации сложной логики оркестратора DreamyDraw. Используется в текущей сессии до завершения Этапа 5.

---

## 🛠 Методология реализации (Step-by-Step)

Для каждого шага пайплайна применяется следующий цикл:
1.  **Реализация:** Написание кода узла (Node) в `Orchestrator`.
2.  **Mock-тест:** Проверка логики на мок-провайдерах (лимиты не ограничены). Исправление ошибок.
3.  **ИИ-тест:** Проверка на реальных API (GPTunnel). 
    *   *Лимит:* до 5 ошибок/попыток.
    *   *Действие при неудаче:* Резюме проблем и отчет руководителю.
4.  **Синхронизация:** Отчет об успехе в чат и запрос на переход к следующему шагу.

## 📝 Правила кодинга и логирования
- **Масштабируемость:** Каждый шаг — это отдельный метод, готовый стать «Node» в LangGraph.
- **Логирование:** Каждый шаг выводит в консоль: `[STEP] Название | Статус | Краткий результат`.
- **Git:** Команда `git add` выполняется сразу после каждого успешно реализованного и проверенного шага.
