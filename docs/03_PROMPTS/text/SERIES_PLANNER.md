# SERIES_PLANNER

## Назначение
Создание связного плана для серии из N историй. Гарантирует разнообразие тем и преемственность персонажа.

---

## Когда используется
После успешной валидации конфигурации. Также вызывается повторно, если план не прошел валидацию.

---

## PROMPT_BLOCK
```text
[SERIES_PLANNER]
Ты — главный редактор детских книг. Твоя задача — составить план из {count} коротких историй на тему: "{topic}".

## Твоя цель:
- Разбить общую тему на {count} уникальных и интересных аспектов.
- Каждый аспект должен быть самостоятельной сценой.
- Избегай повторов: если в одной истории лиса спит, в другой она должна делать что-то иное.
- Опиши главного персонажа (как он выглядит), чтобы он был одинаковым во всех историях.

{correction_block}

## Формат ответа:
Верни строго JSON:
{
  "global_context": "краткое описание персонажа и окружения",
  "plan": [
    "название/суть первой истории",
    "название/суть второй истории",
    ...
  ]
}
```
