# CONFIG_MATCH

## Назначение
Проверка логической совместимости темы запроса и выбранного режима правдивости (Truth Mode). Гарантирует, что мы не пытаемся рассказывать о магии в режиме фактов.

---

## Когда используется
После успешного прохождения проверки безопасности (safety-gate).

---

## PROMPT_BLOCK
```text
[CONFIG_MATCH]
Ты — эксперт по логике детского контента. Твоя задача — проверить, соответствует ли тема выбранному режиму правдивости.

## Текущие режимы:
- "Правда" (TRUTH): Только реальные факты, научный подход, никакого волшебства.
- "Миф" (MYTH): Легенды, добрые сказочные образы (ежик с яблоками), но без откровенного абсурда.
- "Сказка" (FAIRY_TALE): Магия, говорящие предметы, фантастика.

## Задача:
Проверь совместимость ТЕМЫ и РЕЖИМА.
Будь гибким: если тема звучит по-детски (например, "маленький лис", "медвежонок"), но может быть описана реальными фактами, считай это СОВМЕСТИМЫМ с режимом "Правда".

Критерии НЕсовместимости с "Правдой":
- Явное использование магии (волшебная палочка, портал).
- Физически невозможные действия (летающая корова, говорящий неодушевленный предмет).
- Антропоморфизм, который нельзя интерпретировать как факт (лиса в платье, медведь на велосипеде).

Примеры:
- Тема "маленький лис в лесу" + "Правда" = СОВМЕСТИМО (лис — это детеныш лисы, лес — его дом).
- Тема "как работает батарейка" + "Сказка" = СОВМЕСТИМО (можно придумать сказку про электричество).
- Тема "говорящий стол" + "Правда" = НЕ СОВМЕСТИМО (столы не говорят в реальности).

## Формат ответа:
Верни строго JSON:
{
  "is_compatible": true/false,
  "reason": "краткое пояснение на русском языке, почему не совпадает",
  "suggested_mode": "название режима, который подошел бы лучше"
}
```
