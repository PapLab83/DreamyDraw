# DreamyDraw Prompt Documentation Overview

## Назначение документа

Этот раздел документации описывает творческие и смысловые правила генерации контента в DreamyDraw.

Документация является первичным источником правил для:

- генерации коротких детских историй;
- выбора режима правдивости;
- выбора стиля текста;
- генерации иллюстраций;
- проверки качества результата;
- разрешения конфликтов между режимами и стилями.

В DreamyDraw промпты не рассматриваются как отдельные технические шаблоны.  
Они являются частью бизнес-логики продукта.

---

## Главный принцип

DreamyDraw создаёт короткий, добрый и понятный контент для детей 3–5 лет.

Каждый результат должен помогать взрослому начать живой разговор с ребёнком:

- прочитать короткую историю;
- рассмотреть картинку;
- задать простые вопросы;
- обсудить увиденное без перегруза и давления.

DreamyDraw — это не генератор длинных сказок.  
DreamyDraw — это инструмент для короткого совместного взаимодействия ребёнка и взрослого.

---

## Что генерирует система

Для каждой иллюстрации DreamyDraw создаёт:

1. короткий текст;
2. изображение по этому тексту;
3. 2–3 простых вопроса по содержанию.

Текст должен быть пригоден для чтения вслух и для размещения внутри изображения.

---

## Целевая аудитория

Основная аудитория результата — ребёнок 3–5 лет.

Косвенная аудитория — взрослый, который:

- вводит тему;
- выбирает режимы;
- читает текст ребёнку;
- показывает или скрывает картинку;
- задаёт вопросы.

Поэтому текст должен быть понятен ребёнку, но удобен взрослому для чтения.

---

## Основные сущности промптов

Промптовая логика DreamyDraw состоит из нескольких независимых слоёв.

### 1. Базовые инструкции

Базовые инструкции применяются всегда.

Они задают:

- роль ИИ;
- возраст ребёнка;
- язык ответа;
- длину текста;
- формат результата;
- правила вопросов;
- ограничения безопасности.

Файл:

```text
docs/prompts/base/BASE_INSTRUCTION.md
```

---

### 2. Режимы правдивости

Режим правдивости определяет, как устроен мир истории.

Доступные режимы:

- `truth` — реалистичный режим;
- `myth` — мягкий мифологический или культурный образ;
- `fairy_tale` — полностью сказочный режим.

Файлы:

```text
docs/prompts/truth_modes/TRUTH.md
docs/prompts/truth_modes/MYTH.md
docs/prompts/truth_modes/FAIRY_TALE.md
```

---

### 3. Стили текста

Стиль текста определяет интонацию, ритм и лексику.

Доступные стили:

- `gentle` — ласковый;
- `educational` — познавательный;
- `playful` — игровой.

Файлы:

```text
docs/prompts/text_styles/GENTLE.md
docs/prompts/text_styles/EDUCATIONAL.md
docs/prompts/text_styles/PLAYFUL.md
```

---

### 4. Генерация изображений

Промпты изображения описывают:

- композицию;
- стиль иллюстрации;
- возрастную мягкость;
- безопасность;
- читаемость текста внутри картинки;
- визуальные ограничения.

Файлы:

```text
docs/prompts/image/IMAGE_BASE_PROMPT.md
docs/prompts/image/IMAGE_STYLES.md
```

---

### 5. Правила сборки промптов

Финальный промпт собирается из отдельных смысловых блоков.

Для текстовой генерации используется логика:

```text
base instruction
+ selected truth mode
+ selected text style
+ user topic
= final text prompt
```

Для генерации изображения используется логика:

```text
image base prompt
+ selected image style
+ generated story text
= final image prompt
```

Файл:

```text
docs/prompts/composition/PROMPT_COMPOSITION_RULES.md
```

---

### 6. Правила разрешения конфликтов

Некоторые режимы и стили могут конфликтовать.

Например:

- `truth + playful` — игровой стиль не должен заставлять реальных животных разговаривать;
- `myth + educational` — миф нельзя выдавать за научный факт;
- `fairy_tale + educational` — познавательность не должна разрушать сказку.

Файл:

```text
docs/prompts/composition/CONFLICT_RULES.md
```

---

### 7. Проверка качества

Качество результата проверяется по отдельным критериям:

- соответствует ли текст возрасту 3–5 лет;
- соблюдён ли режим правдивости;
- чувствуется ли выбранный стиль;
- нет ли пугающих или неподходящих элементов;
- можно ли ответить на вопросы по тексту;
- подходит ли текст для размещения на картинке.

Файлы:

```text
docs/prompts/evaluation/TEST_CASES.md
docs/prompts/evaluation/QUALITY_CHECKLIST.md
```

---

## Приоритеты инструкций

Если разные инструкции конфликтуют, используется следующий порядок приоритета:

1. безопасность ребёнка;
2. возрастная уместность для 3–5 лет;
3. выбранный режим правдивости;
4. обязательный формат ответа;
5. выбранный стиль текста;
6. художественная выразительность.

Это значит, что стиль не может отменить безопасность или режим правдивости.

---

## Общие правила для всего контента

В любом режиме и стиле DreamyDraw должен:

- писать на русском языке;
- использовать короткие фразы;
- избегать страшных сцен;
- избегать насилия, крови, смерти, угроз и наказаний;
- не использовать сарказм и взрослую иронию;
- не перегружать ребёнка фактами;
- не задавать вопросы, ответа на которые нет в тексте;
- сохранять доброжелательный тон.

---

## Что считается хорошим результатом

Хороший результат:

- короткий;
- понятный ребёнку;
- добрый;
- визуально представимый;
- соответствует выбранному режиму;
- соответствует выбранному стилю;
- содержит 2–3 простых вопроса;
- помогает взрослому начать разговор с ребёнком.

---

## Что считается плохим результатом

Плохой результат:

- слишком длинный;
- слишком взрослый;
- пугающий;
- морализаторский;
- фактически неверный в режиме `truth`;
- хаотичный в игровом стиле;
- выдаёт миф за факт;
- содержит вопросы не по тексту;
- плохо подходит для размещения на картинке.

---

## Принцип расширения

Новые режимы, стили и миры должны добавляться через новые документы или новые разделы документации, не нарушая существующую структуру.

Примеры будущих расширений:

- новые миры: город, организм, английский язык, мифы;
- новые стили текста: поэтический, минималистичный, терапевтический;
- новые стили изображения: карандашный, аппликация, книжная гравюра.

Каждое расширение должно явно описывать:

- назначение;
- что можно;
- что нельзя;
- примеры хорошего результата;
- конфликтные случаи.
